# 代码导览

这份导览按照面试时的讲解顺序整理。七个脚本均来自项目实际开发过程，没有为了展示重新拆分成多层框架。

## 1. `new_scra.py`：任务主程序

这是系统总调度脚本，主要完成：

- 订阅二维码任务序列和 YOLOv5 识别结果；
- 发布 `/stop_flag` 与 `/direction`，给底盘航向控制节点发送运动意图；
- 发布 `/arm_cmd`，按区域和左右位置选择机械臂动作；
- 执行 A 区顺序采摘和 C 区拓扑点位任务；
- 根据识别类别决定跳过、采摘或移除，并播放对应语音。

`PlanC.point_map` 将任务编号映射为 `(road, row)`。`PlanC.action()` 比较当前点与目标点的通道、行号关系，决定同通道移动还是跨通道换道。

## 2. `aicar_pid_line_turn.py`：IMU 航向控制

输入：

- `/imu_data`：IMU 四元数；
- `/stop_flag`：前进、后退、停止、转向等状态；
- `/direction`：相对于初始方向的直行、左转或右转目标。

输出：

- `/cmd_vel`：底盘线速度和角速度。

实现要点：

1. 启动时采样多帧 yaw，计算初始航向 `base_yaw`。
2. 用 `base_yaw +/- pi/2` 表示左右方向。
3. 将角度误差归一化到 `[-pi, pi]`，处理 180 度边界。
4. PID 输出经过限幅和渐变处理后发布到底盘。
5. 停车后继续做小角度对齐，减少下一段路径的累计误差。

## 3. `new_arm.py`：机械臂驱动

脚本从 ROS 参数服务器读取串口、速度、加速度和预设轨迹，订阅 `/arm_cmd`。主程序只发送动作编号，驱动节点查找对应轨迹并逐点发送 JSON 串口命令。

这种方式适合目标位置相对固定的比赛场景：识别位、采摘位、移除位和复位位可以提前现场调好，比赛时不需要实时求逆运动学。

## 4. `Qr_detect.py`：二维码任务读取

处理流程为灰度化、CLAHE 局部对比度增强、自适应阈值、`pyzbar` 解码，再将文本或数字任务转换为 `Int32MultiArray` 发布。该脚本用于处理现场光照不均和二维码反光问题。

## 5. 辅助脚本

- `get_scan_data.py`：从 `/scan` 中提取指定方向附近的最小距离，用于雷达数据验证。
- `navigation_test_goal.py`：依次向 `move_base` 发送多个目标点，验证导航参数和地图坐标。
- `small_arm_driver.py`：通过键盘调节关节角度，也包含根据视觉框中心进行对位的实验代码，主要用于现场机械臂调试。

## 阅读时需要知道的现场版本差异

- 主程序按 `Int32MultiArray` 使用视觉类别和框中心；部分车端视觉桥接版本只发布类别编号。
- `Qr_detect.py` 发布 `/fruit_array`，而该版 `new_scra.py` 订阅 `/Qr_scan`。比赛现场存在不同二维码节点版本，公开仓库保留了各自原始接口。
- 代码中有硬件串口、音频和 DeepStream 输出文件的绝对路径，移植时需要按设备调整。
